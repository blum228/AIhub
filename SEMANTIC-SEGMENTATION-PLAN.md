## ДИАГНОЗ

**Текущая проблема:**  
Все сервисы выглядят одинаково: "AI-девушка с чатом, голосом, NSFW". Пользователь не понимает разницу между Candy.AI и Character.AI, хотя это КАРДИНАЛЬНО разные продукты для разных целей.

**Причина:**  
Сегментация по функциям (голос, NSFW, цена), а не по смыслу (зачем пришёл пользователь).

**Решение:**  
4-уровневый "Смысловой Каркас" — как у Airbnb, Steam, AppStore.

### Уровень 4: ПРОДУКТ-ФИТ (для кого подходит)

Это то, чего НЕТ ни у одного агрегатора. Мы будем первыми.

Для каждого сервиса добавляем:
- **Идеален для:** (3-5 пунктов)
- **Не подходит если:** (2-3 пункта)
- **Альтернативы:** (ссылки на другие сервисы)

**Пример для Candy.AI:**
```yaml
productFit:
  idealFor:
    - "Хочешь генерировать фото своей AI-девушки"
    - "Важна интенсивная эмоциональная вовлечённость"
    - "Ищешь долгосрочные отношения с ботом"
    - "Нравится softcore NSFW + романтика"
    - "Готов платить за качество"
  notFor:
    - "Нужен русский язык"
    - "Ищешь бесплатное решение"
    - "Хочешь hardcore NSFW"
  alternatives:
    - slug: character-ai
      reason: "Если нужно бесплатно"
    - slug: muah-ai
      reason: "Если нужен hardcore NSFW"
    - slug: replika
      reason: "Если нужен русский язык"
```

**Пример для Character.AI:**
```yaml
productFit:
  idealFor:
    - "Фанат персонажей из сериалов/аниме"
    - "Любишь быструю РП уличного стиля"
    - "Хочешь попробовать бесплатно без ограничений"
    - "Важно разнообразие персонажей"
  notFor:
    - "Нужен NSFW контент"
    - "Хочешь глубокие эмоциональные отношения"
  alternatives:
    - slug: candy-ai
      reason: "Если нужен NSFW"
    - slug: replika
      reason: "Если нужна эмоциональная связь"
```

